Objectives:

This is a pretty big undertaking and the goals for this project are fivefold. First and foremost, I want to create a tool that will be useful moving forward in my career as a developer. No project is without bugs, and it's much easier to squash them all with a solid way to track them. It should also prove to be a nice challenge for my skills as a newer developer providing a lot of room for growth and learning.

Speaking of growth and learning, I did mention that there were five goals and only talked about one. The other four are quite simple, yet possibly more important than the first - practice, practice, practice and practice. I want to push myself with this project, and a great way to do that is to incorporate a few technologies that I've yet to utilize. For this project I will be upgrading my skills by going from .NET to the basics of .NET core. I also plan to try out Auth0 for security as opposed to using Identity framework again, and to make it really spicy I'm going to make it a full-stack project and learn a bit of React JavaScript to make a fast, user-friendly interface to top it all off.

Projected Challenges:

There are quite a few with this one, but that's what makes it so exciting. As I learn to create more and more through computer programming, I need to start focusing on making apps that have a clear organization and are scalable. So with this one, I'm designing this app as though in the future I'm not the only programmer who's going to work on it. In order to do that, I need to do some research into creating SOLID applications and look into incorporating various useful design patterns. Here are a couple of resources I've been using to get started:

  • C# Design Patterns : Factory Method
  • C# Builder Design Pattern

In addition to design considerations, I'm going to need some pre-practice on using the new technologies I intend to include in this project. I've familiarity with .NET, but it will be the first time for me to use .NET core instead. A friend of mine recommended this book to get me started:

  • C# 7.1 and .NET Core 2.0 - Modern Cross-Platform Development

Although we didn't cover .NET core during my time at Grand Circus, we were given that exact book for extra study - perfect!

Though I've some experience with JavaScript to help get me started learning React, I have no idea what's involved in using Auth0. It's going to be a completely brand new adventure! But that's what these projects are for, so without further ado, I present the resources I will be scouring to get myself up to speed enough to complete my small project. If you have any more resources or advice for this type of project, hit up the e-mail page and shoot a message my way!
